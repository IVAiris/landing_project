# landing-form-training

Это учебный локальный проект на Next.js. Лендинг с одной формой.

Сам проект (код) лежит в `training/my-app` — все npm-команды нужно запускать
из этой папки (`cd training/my-app`), а не из `training/` или корня репозитория.

## Текущее состояние (реализовано)

- Лендинг с секциями `hero`, `плюсы`, `вопросы`, `форма`.
- Форма с полями `имя`, `email`, `описание задачи`.
- Клиентское состояние формы и клиентская проверка: пустые поля, формат email.
- Сообщения об ошибках у полей и сообщение об успехе при корректном заполнении.
- API-роут `POST /api/requests` (`src/app/api/requests/route.ts`): принимает JSON
  с полями `name`, `email`, `description`, проверяет их серверной Zod-схемой
  (`src/lib/validation/requestSchema.ts`).
  - Если данные не прошли проверку — ответ `400` с полем `errors` (понятные
    сообщения по каждому полю).
  - Если данные корректны — временный успешный ответ
    `{ message: "API-роут работает", received: {...} }`. Пока без сохранения
    данных; форма на клиенте на этот роут ещё не отправляет запрос.
- Серверная Zod-проверка подключена, без SQLite, без Tailwind.

## Roadmap (следующий урок, ещё не реализовано в коде)

- Локальное сохранение заявок в SQLite.
- Отправка формы на клиенте на `POST /api/requests` (сейчас форма работает только локально).

Эти пункты — цель следующего этапа курса. В `my-app` они пока не подключены;
`sqlite3` может быть добавлен в зависимости заранее, но использоваться
в коде начнёт только когда до этого дойдёт урок.
