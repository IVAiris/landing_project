# landing-form-training

Это учебный локальный проект на Next.js. Лендинг с одной формой.

Сам проект (код) лежит в `training/my-app` — все npm-команды нужно запускать
из этой папки (`cd training/my-app`), а не из `training/` или корня репозитория.

## Текущее состояние (реализовано)

- Лендинг с секциями `hero`, `плюсы`, `вопросы`, `форма`.
- Форма с полями `имя`, `email`, `описание задачи`.
- Клиентское состояние формы и клиентская проверка: пустые поля, формат email.
- API-роут `POST /api/requests` (`src/app/api/requests/route.ts`): принимает JSON
  с полями `name`, `email`, `description`, проверяет их серверной Zod-схемой
  (`src/lib/validation/requestSchema.ts`).
  - Если данные не прошли проверку — ответ `400` с полем `errors` (понятные
    сообщения по каждому полю).
  - Если данные корректны — данные сохраняются в локальную SQLite-базу
    (`data/app.db`, таблица `requests`, поля `name`, `email`, `description`,
    `created_at`) через `src/lib/db.ts`, и возвращается успешный ответ
    `{ message: "Заявка сохранена" }`.
- Форма на клиенте (`src/app/page.tsx`) после успешной клиентской проверки
  отправляет `POST /api/requests` с полями `name`, `email`, `description`.
  - Пока запрос выполняется, кнопка отправки блокируется и показывает
    состояние отправки.
  - При успешном ответе показывается сообщение об успехе.
  - При ответе `400` серверные сообщения об ошибках по каждому полю
    (`errors.name/email/description`) выводятся под соответствующими полями.
  - При других ошибках (сеть, `500` и т.п.) показывается общее сообщение
    об ошибке отправки.
  - Введённые данные не сбрасываются при ошибке.
- Серверная Zod-проверка и сохранение в SQLite подключены, без Tailwind.

## Стек и конфигурация

- Next.js;
- React;
- TypeScript;
- npm;
- App Router;
- папка `src`;
- обычный CSS без Tailwind;
- SQLite (пакет `sqlite3`) для локального хранения заявок.

## Основные команды

- `cd training/my-app && npm run dev` — запуск проекта локально.
- `cd training/my-app && node -e "const s=require('sqlite3');const db=new s.Database('data/app.db');db.all('SELECT * FROM requests',[],(e,r)=>{console.log(r);db.close();});"` —
  посмотреть сохранённые заявки без установки дополнительных инструментов.
